# PRD — 제품 요구사항 정의서

## 1. 제품 개요

### 1.1 제품명
Kanban Board (칸반보드)

### 1.2 버전
v2.0.0

### 1.3 제품 비전
작업 항목을 시각적으로 관리하고 진행 상태를 한눈에 파악할 수 있는 경량 칸반보드.
Google / GitHub / 이메일 계정으로 로그인하며, 내 보드 데이터는 계정별로 분리 관리된다.
GitHub Pages에 배포되어 별도 설치 없이 어디서나 접근 가능하다.

**배포 URL**: https://hyunhong93.github.io/kanban-board

### 1.4 목표
- 작업 항목(카드)을 To-do → In-Progress → Done 흐름으로 직관적으로 관리
- 드래그 앤 드롭으로 카드 이동 시 마찰 최소화
- 새로고침 후에도 상태 유지
- Google / GitHub / 이메일로 간편하게 로그인
- 계정별 독립된 보드 데이터 유지

## 2. 대상 사용자

| 페르소나 | 설명 |
|---|---|

## 3. 핵심 기능 요구사항

### 3.1 칸반 보드 (FR-01)

| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-01-1 | 화면에 To-do / In-Progress / Done 3개 컬럼을 표시한다 | Must |
| FR-01-2 | 각 컬럼은 헤더(아이콘 + 이름 + 카드 수 뱃지)를 갖는다 | Must |
| FR-01-3 | 컬럼은 화면 너비에 따라 3열(≥700px) 또는 1열(<700px)로 배치된다 | Should |

### 3.2 카드 관리 (FR-02)

| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-02-1 | 상단 입력 폼에서 카드 텍스트와 대상 컬럼을 선택해 카드를 추가할 수 있다 | Must |
| FR-02-2 | Enter 키 또는 추가 버튼으로 카드를 추가할 수 있다 | Must |
| FR-02-3 | 공백만으로 구성된 텍스트는 카드 추가를 거부한다 | Must |
| FR-02-4 | 카드에 마우스를 올리면 삭제 버튼(X)이 나타난다 | Must |
| FR-02-5 | 삭제 버튼 클릭으로 카드를 삭제할 수 있다 | Must |
| FR-02-6 | 컬럼이 비어있을 때 안내 텍스트("카드를 드래그하거나 추가하세요")를 표시한다 | Should |

### 3.3 드래그 앤 드롭 (FR-03)

| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-03-1 | 카드를 드래그해 다른 컬럼으로 이동할 수 있다 | Must |
| FR-03-2 | 카드를 같은 컬럼 내에서 위아래로 순서를 변경할 수 있다 | Must |
| FR-03-3 | 드래그 중 원본 카드는 반투명하게 표시된다 | Should |
| FR-03-4 | 드래그 중 대상 카드의 위/아래에 삽입 위치 인디케이터(색상 보더)가 표시된다 | Should |
| FR-03-5 | 드래그 중 대상 컬럼에 강조 효과(아웃라인 glow)가 표시된다 | Should |
| FR-03-6 | 빈 컬럼에도 카드를 드롭할 수 있다 | Must |

### 3.4 상태 지속성 (FR-04)

| ID | 요구사항 | 우선순위 |
|---|---|---|
| FR-04-1 | 카드 상태(텍스트, 컬럼, 순서)를 브라우저 localStorage에 저장한다 | Must |
| FR-04-2 | 페이지 새로고침 후 마지막 상태가 복원된다 | Must |
| FR-04-3 | 최초 실행 시(localStorage 없음) 샘플 카드 4개를 표시한다 | Could |

### 3.5 인증 (FR-05) — v2.0 신규

| ID | 요구사항 | 우선순위 |
|---|---|---|

| FR-05-5 | 로그인 후 헤더에 사용자 이름(또는 아바타)이 표시된다 | Should |

| FR-05-8 | 계정별로 독립된 카드 데이터를 유지한다 (localStorage 키 스코핑) | Must |

## 4. 비기능 요구사항

| ID | 요구사항 | 기준 |
|---|---|---|
| NFR-01 | 핵심 JS 라이브러리 최소화 (Supabase SDK CDN만 추가) | CDN 1개 |
| NFR-02 | 카드 추가/삭제/이동 후 화면 갱신 지연 없음 | 즉각 반응 |
| NFR-03 | 모바일/태블릿 화면에서 레이아웃 유지 | 700px 이하 대응 |
| NFR-04 | 접근성 — 주요 버튼에 aria-label 제공 | 스크린리더 기본 지원 |
| NFR-05 | GitHub Pages 정적 배포 | 별도 서버 불필요 |

## 5. 제외 범위 (Out of Scope)

- 서버 사이드 저장 (백엔드 / 데이터베이스)
- 카드 상세 정보 (설명, 마감일, 담당자, 첨부파일)
- 카드 검색 / 필터
- 실시간 협업 (WebSocket / Supabase Realtime)
- 알림 / 이메일 알림
